# Stakeholder Engagement Log Template

## Purpose
Track all stakeholder interactions, commitments, and feedback throughout the project lifecycle. Ensures accountability, maintains communication history, and helps identify patterns in stakeholder needs.

## When to Use
- Throughout all project phases, from initiation to closure
- Before and after key milestones or decision points
- When planning stakeholder communications
- During status reviews and retrospectives

---

## How to Use This Template

1. **Maintain one log per project** in your project repository
2. **Update after each interaction** with stakeholders
3. **Review weekly** with PM to identify follow-up needs
4. **Reference in status reports** to show stakeholder alignment
5. **Archive at project closure** for future reference

| Stakeholder Name | Role/Title | Interest Level | Influence Level | Preferred Communication | Engagement Strategy |
|------------------|------------|----------------|-----------------|-------------------------|---------------------|

**Interest Level**: How much this stakeholder cares about the project (High/Medium/Low)

**Preferred Communication**: Stakeholder's preferred channels and frequency

## Engagement Log Entries

### Entry Template
Use this format for each stakeholder interaction:

---

#### [Date] - [Stakeholder Name] - [Interaction Type]

**Type**: [Meeting / Email / Slack / Demo / Review / Decision / Other]  
**Attendees**: [Who was present]  
**Purpose**: [Why this interaction occurred]

**Discussion Summary**:
- [Key points discussed]
- [Decisions made]
- [Concerns raised]

**Action Items**:
- [ ] [Action item 1] - Owner: [Name] - Due: [Date]
- [ ] [Action item 2] - Owner: [Name] - Due: [Date]

**Follow-up Required**: [Yes/No] - [When and what]  
**Sentiment**: [Supportive / Neutral / Concerned / Blocking]  
**Notes**: [Additional context or observations]

---

### Example Entry

---

#### 2026-01-05 - Jane Smith (VP Engineering) - Planning Review

**Type**: Meeting (1:1)  
**Attendees**: Jane Smith (VP Eng), Alex Johnson (PM)  
**Purpose**: Review Q1 release plan and resource allocation

**Discussion Summary**:
- Reviewed proposed timeline for new API feature launch
- Jane concerned about overlap with infrastructure migration project
- Agreed to extend timeline by 2 weeks to reduce team pressure
- Requested weekly progress updates during execution phase

**Action Items**:
- [x] Update release plan with new timeline - Owner: Alex Johnson - Due: 2026-01-07
- [ ] Schedule weekly check-ins for 15 min on Fridays - Owner: Alex Johnson - Due: 2026-01-08
- [ ] Coordinate with infrastructure team on resource sharing - Owner: Jane Smith - Due: 2026-01-12

**Follow-up Required**: Yes - Send updated plan by Jan 7, first weekly check-in on Jan 12  
**Sentiment**: Supportive (with timeline adjustment)  
**Notes**: Jane emphasized importance of quality over speed. Appreciates proactive communication.

## Communication Patterns and Insights

### Lessons Learned
Document patterns and insights for future reference:

**What worked well:**
- Weekly email updates kept stakeholders informed without meeting overload
- Early involvement of Sales in planning phase reduced late-stage changes
- Monthly demos increased stakeholder confidence and engagement

**What could be improved:**
- Some stakeholders prefer Slack for quick questions vs. email
- Need to engage Finance earlier for budget discussions
- Technical stakeholders want more detailed sprint updates

**Recommendations for next project:**
- Create stakeholder-specific communication plans during initiation
- Use different update formats for technical vs. business stakeholders
- Schedule "office hours" for ad-hoc stakeholder questions

## Maintenance Guidelines

**Weekly**:
- Add new engagement log entries for all stakeholder interactions
- Review and update outstanding action items
- Update sentiment tracking if changes observed

**Monthly**:
- Review engagement frequency for each stakeholder
- Identify stakeholders who haven't been contacted recently
- Update communication strategies based on what's working

**At Milestones**:
- Generate summary of stakeholder engagement for that phase
- Document lessons learned about communication effectiveness
- Adjust engagement approach for next phase if needed

**At Project Closure**:
- Complete final summary of all stakeholder interactions
- Document overall communication effectiveness
- Archive log for future project reference
